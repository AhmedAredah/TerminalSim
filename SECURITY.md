# Security Policy

## Supported Versions

TerminalSim is under active development. Security updates are provided for the
current stable release line and the default branch.

| Version | Supported |
| ------- | --------- |
| `main` | Yes |
| `1.x` | Yes |
| `< 1.0` | No |

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues.

Use GitHub Security Advisories for private vulnerability reports:
[https://github.com/AhmedAredah/TerminalSim/security/advisories/new](https://github.com/AhmedAredah/TerminalSim/security/advisories/new)

If you cannot use GitHub Security Advisories, email
[AhmedAredah@vt.edu](mailto:AhmedAredah@vt.edu) with:

- A description of the vulnerability and affected components
- Steps to reproduce or a proof of concept, if available
- The potential impact
- Any suggested mitigation or patch

## Response SLA

The maintainer will make a best effort to:

- Acknowledge the report within 2 business days
- Provide an initial assessment within 7 business days
- Share status updates at least every 14 days until resolution
- Coordinate disclosure timing with the reporter when a fix is available

Security fixes may be released as patches, documented mitigations, or both,
depending on severity and affected versions.
